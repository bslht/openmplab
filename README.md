# openmplab
В программу добавлены директивы для распараллеливания циклов, а также реализована возможность задания количества потоков через аргументы командной строки.
Были проведены эксперименты для матриц размером от 200×200 до 2000×2000 с использованием 1, 2, 4 и 8 потоков. Для проверки корректности работы параллельной программы было выполнено сравнение результатов, полученных с помощью OpenMP, с эталонными значениями, вычисленными с использованием библиотеки NumPy. Верификация проводилась для всех размеров матриц (200, 400, 800, 1200, 1600, 2000) и для всех количеств потоков (1, 2, 4, 8). В ходе экспериментов было получено ускорение для различных размеров матриц и количества потоков. Для 2 потоков ускорение составило 1.97–2.16x (эффективность 98–108%), для 4 потоков – 3.67–3.88x (эффективность 92–97%), для 8 потоков – 3.33–6.15x. Наилучший результат достигнут для матрицы 1600×1600 на 8 потоках – **6.15x**. Для маленьких матриц (200×200) ускорение ограничено накладными расходами OpenMP, для больших матриц (1200–2000) наблюдается эффективное масштабирование. Результаты подтверждают корректность параллельной реализации и эффективность использования OpenMP для данной задачи.
